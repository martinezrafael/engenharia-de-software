_Definição Formal_
Uma API (Application Programming Interface) é um conjunto de rotinas e padrões estabelecidos por um software para a utilização das suas funcionalidades por aplicativos que não pretendem envolver-se em detalhes da implementação do software, mas apenas usar seus serviços.

Uma API é um intermediário, uma interface entre o que você quer e como realizar o que você quer

Basicamente é alguma forma de você ter acesso a algum consjunto de serviços

## 🔌 O que é uma API?

API significa Application Programming Interface (Interface de Programação de Aplicação).

Na prática, ela é um ponto de contato. É a forma como um software "abre as portas" para que outro software use suas funcionalidades sem precisar entender como ele foi escrito por dentro.

_A Analogia do Restaurante_

1. Você (Cliente): Quer comer, mas não sabe cozinhar ou não tem acesso à cozinha.
2. O cardápio(API): Lista o que você pode pedir e o que receberá em troca.
3. O Garçom (API): Leva seu pedido até a cozinha e traz o prato pronto. Você não precisa saber se o chef usou fogão a gás ou indução (Detalhes de implementação), você só precisa do resultado.

## 📂 Tipos Comuns de APIs

Nem toda API está na internet. Elas se dividem principalmente em dois mundos:

_1. APIs de Código Fonte (Library/SDK APIs)_
São as APIs que você usa dentro do seu código ao importar uma biblioteca.

- Exemplo: Quando você usa Math.random() no JavaScript ou uma biblioteca para gerar PDFs. Você chama uma função (a interface) e ela te entrega o resultado.

_2. APIs Web (Remote APIs)_
São as mais famosas hoje em dia. A comunicação acontece entre computadores diferentes através de uma rede (Internet), geralmente usando o protocolo HTTP.

---

## 🌐 Como funciona uma API Web?

O fluxo básico segue o modelo Requisição (Request) e Resposta (Response):

1. Request: O seu aplicativo envia um "envelope" com dados (ex: um CEP).
2. Processamento: A API recebe, valida e processa (ex: consulta o banco de dados).
3. Response: A API devolve um "envelope" com o resultado (ex: o endereço completo) ou um erro.

---

## 🗺️ Padrões de APIs Web (Protocolos)

Existem diferentes "idiomas" que as APIs podem falar:

| Padrão      | O que é?                          | Característica Principal                                                               |
| :---------- | :-------------------------------- | :------------------------------------------------------------------------------------- |
| **RPC**     | _Remote Procedure Call_           | Focado em executar uma ação/função em outro servidor como se fosse local.              |
| **SOAP**    | _Simple Object Access Protocol_   | Um padrão rígido e antigo, baseado em arquivos XML. Muito usado em bancos.             |
| **REST**    | _Representational State Transfer_ | **O mais comum hoje.** Focado em "Recursos" (ex: `/usuarios`) e usa os verbos HTTP.    |
| **GraphQL** | _Query Language_                  | O cliente escolhe exatamente quais dados quer receber, evitando excesso de informação. |

## 📋 Resumo

| Conceito      | Descrição                                                                                               |
| :------------ | :------------------------------------------------------------------------------------------------------ |
| **Interface** | O ponto de encontro onde dois sistemas se comunicam.                                                    |
| **Endpoint**  | O endereço (URL) onde a API pode ser acessada (ex: `api.exemplo.com/v1/cep`).                           |
| **Payload**   | Os dados que viajam dentro da requisição ou resposta (geralmente em formato JSON).                      |
| **Agnóstica** | Uma API pode ser feita em Java e ser consumida por um app em Flutter. A tecnologia interna não importa. |
